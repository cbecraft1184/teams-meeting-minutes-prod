That error message, "The value provided for bots[0].scopes[2] is invalid," means you have an unsupported or misspelled value in the scopes list for your bot within your Microsoft Teams app's manifest.json file.The path bots[0].scopes[2] tells you exactly where the problem is:bots[0]: The first (and usually only) bot definition in your manifest's bots array.scopes[2]: The third scope listed in that bot's scopes array (because array indexing starts at 0).âœ… Valid Scope ValuesTeams bots can only be installed in specific contexts, which are defined by these three valid strings. Check your manifest.json and ensure the value at the third position in the scopes array is one of the following, spelled exactly as shown:Valid ScopeDescriptionpersonalA 1:1 chat between the user and the bot (also known as the "private" or "personal" scope).teamA chat within a standard Teams channel.groupChatA multi-person chat that is not a channel.Example of a Correct scopes ArrayIf your bot supports all three installation contexts, your manifest should look like this:JSON{
  "bots": [
    {
      "botId": "YOUR_BOT_ID",
      "scopes": [
        "personal",  // [0]
        "team",      // [1]
        "groupChat"  // [2] <- This is the value you need to check
      ],
      // ... other bot properties
    }
  ]
}
